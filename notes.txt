*linear search->a)best case scenario is when element is found at very 1st position
					->thus the best case efficiency(BCE) of linear search is O(1) Read it as: Big-Oh of One
				b)worst case is when element is not found or at very last position
					->worst case efficiency(WCE) would be O(n) where n is size of the list
				c)if element found somwhere between the list then Average case efficiency would be O(n)
*The solution for linear search is Brute Force Technique.
	->Brute Force Technique is the technique that come usually naturally without having to apply any specific Math rule or formulea.Thus there is no inventor or author for these 
	solutions
				
*Characteristics os Big-Oh:
	1)it is used usually to find WCE. However, we can also find the other 2.
	2)THe Big-Oh always depends on the i/p size and gives the efficiency in order of the i/p size, say 'n'
	3) the Big-Oh is here to tell us the order in which the algorithm is. It's duty is not to give the exact efficiecy of the algorithm.
	4)the Big-Oh gives efficiency in terms of 'n'

	sorted = true  # assume the list is sorted
	for j from 0 to n-2-i do:
		if list[j] > list[j+1]:
			swap list[j], list[j+1]
			sorted = false
	if sorted:
		break

BCE O(n)
